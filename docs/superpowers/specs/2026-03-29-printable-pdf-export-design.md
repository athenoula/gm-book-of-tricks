# Printable PDF Export Design Spec

**Date:** 2026-03-29
**Status:** Draft

## Overview

Add printable PDF generation to Book of Tricks so GMs can download formatted prep sheets, stat blocks, and content bundles for use at the table without a laptop.

## Core Decisions

- **Format:** PDF (via `@react-pdf/renderer`)
- **Themes:** Themed (parchment/fantasy aesthetic) and Clean (ink-friendly, white background)
- **Scope:** Session prep sheets, individual content items, multi-select bundles
- **Architecture:** Client-side PDF generation, direct blob download

## PDF Generation Engine

### Library

`@react-pdf/renderer` — renders React components directly to PDF blobs. No DOM rendering, no browser print dialog. Text is selectable, filenames are controlled, output is consistent.

### Generation Flow

1. User clicks "Print" (or "Print N Items" for bundles)
2. Fetch required data via shared data-fetching utility
3. Pass data to the appropriate PDF React component
4. Call `pdf(<Component />).toBlob()` to generate the PDF blob
5. Download via `URL.createObjectURL` + programmatic `<a>` click (same pattern as ZIP export)

### Shared Data Fetcher

Extract per-table Supabase queries from `campaign-exporter.ts` into a new shared module `src/lib/export/fetch-campaign-data.ts`. Both the markdown export (`campaign-exporter.ts`) and PDF generation (`generate.ts`) import from this shared module.

The shared fetcher provides functions like:
- `fetchCampaignWithCounts(campaignId)` — campaign row + entity counts
- `fetchSessionWithTimeline(sessionId)` — session + timeline blocks (ordered by sort_order)
- `fetchPCsWithDetails(campaignId)` — PCs + their spells and inventory
- `fetchNPCsWithSpells(campaignId)` — NPCs + their spells
- `fetchMonsters(campaignId)` — all monsters
- `fetchSpells(campaignId)` — all spells
- `fetchItems(campaignId)` — all items
- `fetchLocations(campaignId)` — all locations with hierarchy
- `fetchHandouts(campaignId)` — all handouts
- `fetchBattles(campaignId)` — all battles
- `fetchResolvedTimelineContent(blocks)` — given timeline blocks, fetch the full content for any referenced monsters/NPCs/spells/locations by their `source_id`

## PDF Themes

### Themed

- Background: warm parchment (#f5f0e8)
- Headings: Cinzel-like serif (use a standard serif since @react-pdf/renderer needs registered fonts — register Cinzel and Crimson Pro from Google Fonts)
- Body: Crimson Pro
- Accents: gold-brown tones (#c4a87c, #5c3d2e)
- Dividers: ornamental horizontal rules
- Stat block cards: subtle background with gold-toned borders
- Recap: styled callout with left border accent

### Clean

- Background: white
- Headings: bold system serif (Georgia or Times)
- Body: system serif
- Accents: black/grey
- Dividers: simple 1px lines
- Stat block cards: plain bordered boxes
- Recap: left-bordered block, grey

Both themes share the same layout structure — only colors, fonts, and decorative elements differ. Implemented as two style objects in `styles.ts`, selected by a `theme` prop.

## PDF Components

### File Structure

```
src/lib/export/pdf/
├── styles.ts             — themed + clean StyleSheet objects
├── primitives.ts         — reusable building blocks
├── SessionPrepPDF.tsx    — session prep sheet
├── MonsterPDF.tsx        — monster stat block
├── NPCPDF.tsx            — NPC reference
├── SpellPDF.tsx          — spell card
├── PCPDF.tsx             — character summary
├── ItemPDF.tsx           — item card
├── LocationPDF.tsx       — location summary
├── HandoutPDF.tsx        — handout content
├── BattlePDF.tsx         — encounter table
├── BundlePDF.tsx         — multi-item wrapper
└── generate.ts           — orchestrator functions
```

### Primitives (`primitives.ts`)

Shared building blocks used across all PDF components:

- `PageHeader` — document title, campaign name, date
- `SectionHeading` — styled heading with optional divider
- `Divider` — ornamental (themed) or plain (clean) horizontal rule
- `StatBlockCard` — compact card: name, AC/HP/CR row, key abilities/actions summary
- `SpellCard` — compact spell reference: level, school, casting time, range, brief description
- `NPCCard` — compact NPC: name, personality snippet, key stats
- `AbilityScoreRow` — STR/DEX/CON/INT/WIS/CHA inline row
- `PageFooter` — page number, "Generated by Book of Tricks"

Each primitive accepts a `styles` prop (from the theme).

### SessionPrepPDF

The main document. Structure:

1. **Header:** session name, number, campaign, date
2. **Recap:** previous session recap in a callout block
3. **Session notes:** if any
4. **Timeline:** walks through timeline blocks in sort order:
   - `scene`/`note` — heading + content text
   - `battle` — heading + grid of `StatBlockCard` for each combatant (resolved from content_snapshot or fetched by source_id)
   - `monster` — inline `StatBlockCard`
   - `npc` — inline `NPCCard` with stats and spells
   - `spell` — inline `SpellCard`
   - `location` — name + description
   - `handout` — heading + content preview

Data needed: session row, timeline blocks (sorted), and for each block that references content: the full monster/NPC/spell/location record. Use `fetchResolvedTimelineContent(blocks)` to bulk-fetch all referenced content.

### Individual Content PDFs

Each is a simple `<Document><Page>` containing the formatted content:

- **MonsterPDF** — full stat block: name, size/type/alignment, AC, HP, speed, ability scores, actions, special abilities, legendary actions
- **NPCPDF** — name, race/occupation, personality, appearance, stats, spells, notes
- **SpellPDF** — name, level/school, casting time, range, components, duration, description, at higher levels
- **PCPDF** — name, class/level/race, ability scores table, HP/AC/speed, equipment, spells, features, traits
- **ItemPDF** — name, type/rarity, description, properties
- **LocationPDF** — name, type, description, parent/child locations, notes
- **HandoutPDF** — name, template type, content
- **BattlePDF** — name, combatant table (name, HP, AC, initiative)

### BundlePDF

Wraps multiple items into one document. Each item starts on a new page. Structure:

```tsx
<Document>
  <Page>Table of Contents (if > 5 items)</Page>
  <Page>Item 1</Page>
  <Page>Item 2</Page>
  ...
</Document>
```

The bundle can contain mixed content types (e.g., 3 monsters + 2 spells). Each item renders using its corresponding PDF component's inner content (not wrapped in its own `<Document>`).

### Generate Orchestrator (`generate.ts`)

Public functions:

```typescript
generateSessionPrepPDF(sessionId: string, campaignId: string, theme: 'themed' | 'clean'): Promise<void>
generateMonsterPDF(monster: Monster, theme: 'themed' | 'clean'): Promise<void>
generateNPCPDF(npc: NPC, spellNames: string[], theme: 'themed' | 'clean'): Promise<void>
generateSpellPDF(spell: Spell, theme: 'themed' | 'clean'): Promise<void>
generatePCPDF(pc: PlayerCharacter, spellNames: string[], inventory: string[], theme: 'themed' | 'clean'): Promise<void>
generateItemPDF(item: Item, theme: 'themed' | 'clean'): Promise<void>
generateLocationPDF(location: Location, allLocations: Location[], theme: 'themed' | 'clean'): Promise<void>
generateHandoutPDF(handout: Handout, theme: 'themed' | 'clean'): Promise<void>
generateBattlePDF(battle: Battle, theme: 'themed' | 'clean'): Promise<void>
generateBundlePDF(items: Array<{ type: string; data: unknown }>, theme: 'themed' | 'clean'): Promise<void>
```

For items that need fetching (e.g., `generateSessionPrepPDF`), the function fetches data internally. For items where data is already available in the component (e.g., `generateMonsterPDF` called from a page that already loaded the monster), data is passed directly.

Each function: builds the React component → calls `pdf().toBlob()` → triggers download with a descriptive filename (e.g., `young-red-dragon-stat-block.pdf`, `session-03-prep.pdf`).

## UI Integration

### Individual Print Buttons

Added to existing pages alongside other action buttons:

| Page | Button Location | Label |
|------|----------------|-------|
| Session page (SessionPage) | Session header, next to status | "Print Prep Sheet" |
| Bestiary (BestiaryPage) | Each monster card/detail view | Print icon button |
| Spellbook (SpellbookPage) | Each spell row/card | Print icon button |
| Characters (CharactersPage/PCSheet) | Character header | "Print" button |
| Locations (LocationsPage) | Each location card | Print icon button |
| NPC details (where displayed) | NPC header/card | Print icon button |

Each print button includes a small theme selector — either a dropdown or a simple toggle. Default theme: themed.

### Multi-Select Mode

Available on: **BestiaryPage, SpellbookPage, CharactersPage, LocationsPage**

**Activation:** "Select" toggle button in the page header (next to search/filters).

**When active:**
- Checkboxes appear on each item card (left edge)
- A floating bottom bar slides up with:
  - Left: "N items selected" count
  - Center: theme toggle (Themed / Clean, styled as segmented control)
  - Right: "Print N Items" button (primary) + "Cancel" (ghost)
- Clicking an item toggles its selection
- "Print N Items" generates a `BundlePDF` and downloads

**Print Select Store** (`usePrintSelectStore`):

```typescript
interface PrintSelectStore {
  active: boolean
  selectedIds: Set<string>
  contentType: 'monster' | 'spell' | 'character' | 'location'
  theme: 'themed' | 'clean'
  enterSelectMode: (contentType: string) => void
  exitSelectMode: () => void
  toggle: (id: string) => void
  selectAll: (ids: string[]) => void
  clearSelection: () => void
  setTheme: (theme: 'themed' | 'clean') => void
}
```

Zustand store, lives at `src/lib/export/pdf/usePrintSelectStore.ts`. Selection is cleared when navigating away or exiting select mode.

### Floating Print Bar Component

`src/components/ui/PrintSelectionBar.tsx` — a fixed-bottom bar (z-50, above content, below toasts) that shows when `usePrintSelectStore.active` is true. Contains the item count, theme toggle, and action buttons. Slides up/down with motion animation.

## Design Notes

- Session prep PDF includes a recap section. If the session has its own recap field populated, use that. If not (e.g., upcoming session), fetch the previous session (by session_number) and use its recap to give the GM context on where the party left off.
- For timeline blocks with `source_id`, fetch the full record to get complete stat blocks. For blocks without `source_id` (inline content), use the `content_snapshot` data.
- PDF filenames use slugified names: `session-03-the-dragons-lair-prep.pdf`, `young-red-dragon.pdf`, `bestiary-bundle-5-items.pdf`
- Toast feedback: "Generating PDF..." on start, "PDF downloaded" on success, error toast on failure
- Multi-select bar respects existing z-index hierarchy: z-50 (same as modals, below toasts at z-70)
